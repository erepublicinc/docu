software layers

The software needs to be layered as follows
  • page layer
  • website layer, code for the specific content, css etc
  • core layer, very generic
  • php on apache , memcache? , database(whichever we choose)

why layers?

thinking about the code as being in layers gives us the following advantages:
- Different levels of testing, for example a core-level change has to go through a lot more testing
- Code review, for example core-level change has to have a code review
- Documentation, for example core-level change has to have the documentation changed, and this should be qa'ed as well'

page layer

this layer deals with individual pages, changes will opnly affect a single page

website layer

this layer deals with the websites, changes will affect a website. for example the css files for a site fit into this category
